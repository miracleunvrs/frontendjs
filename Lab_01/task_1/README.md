# Отчет по Lab 1.1: Сравнение императивного и декларативного подходов в React

</div>

В данной работе были реализованы два подхода к созданию пользовательского интерфейса:
* **Императивный подход** с ипользованием библиотеки JQuery.
* **Декларативный подход** с использованием библиотеки React.

Основное различие между ними заключается в том, как приложение управляет обновлением интерфейса при изменении данных.

## Императивный подход (JQuery)
В императивном подходе разработтчик описывает "как" должно произойти изменение. Код проедставялет собоф последовательность инструкций: 
* Найти элемент в DOM
* Прослушать событие клика и вручную изменить класс элемента с помощью метода `.toggleClass()`.

Как отмечается в Главе 1 учебника, в разделе "Declarative UI structures", такой подход заставляет программиста вручную отслеживать все связи между данными и элементами урпавления. Это неизбежно ведет к усложению кода, который становится трудно поддерживать и отлаживать по мере роста приложения.

## Декларативный подход (React)
Напротив, React использует декларативный подход, где разработчик описывает "что" должно быть отображено на основе текущего состояния данных. В этом случае:
* Компонент `ToggleButton` принимает состояние `isHighlighted` и функцию `onClick`.
* React автоматически обновляет DOM при изменении состояния, что упрощает управление интерфейсом.
Как указано в учебнике, декларативный стиль позволяет сосредоточиться на логике отображения, а не на деталях манипуляции DOM. Это приводит к более чистому и понятному коду, который легче тестировать и поддерживать.

## Сравнение и преимущества декларативного подхода
Декларативный подход масштабируется значительно лучше императивного. В сложных интерфейсах с сотнями элементов ручное управление каждым изменением в jQuery привело бы к созданию запутанного кода («spaghetti code»). React решает эту проблему, делая интерфейс предсказуемым: UI — это просто функция от состояния. Использование виртуального DOM (Virtual DOM) позволяет React эффективно вычислять разницу между старым и новым состоянием, обновляя только необходимые части страницы, что повышает производительность и упрощает архитектуру крупных проектов.

## Вывод
В заключение, переход от императивного управления к декларативному описанию состояний — это ключевой шаг к созданию современных, поддерживаемых и высокопроизводительных веб-приложений.
