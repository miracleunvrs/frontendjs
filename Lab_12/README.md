## Responsive Design Implementation (Объяснение адаптивности)

В вебе адаптивность достигается за счет CSS Media Queries (`@media`). В React Native используется программный подход:

1. **Размеры экрана (`useWindowDimensions`)**:
   Мы получаем текущую ширину и высоту экрана с помощью хука `useWindowDimensions()`. Это позволяет нам программно определять тип устройства:
   - `const isTablet = width >= 768;`
   - `const isLandscape = width > height;`
   Опираясь на эти переменные, мы динамически подменяем стили (например, рендерим сайдбар для планшета и обычный список для телефона).

2. **Работа с Safe Area (`react-native-safe-area-context`)**:
   Чтобы интерфейс не перекрывался "челкой" (Notch) на современных iPhone или статус-баром на Android, используется хук `useSafeAreaInsets()`. Он возвращает точные значения отступов (top, bottom, left, right), которые мы добавляем к `padding` наших контейнеров (например, в кастомном `ResponsiveHeader`).

3. **Сетка (Grid Layout)**:
   Так как в React Native нет нативного CSS Grid, сетка рассчитывается математически. Мы берем ширину экрана, вычитаем отступы (spacing) и делим на количество нужных колонок, получая точный `width` для карточек. А за перенос на новую строку отвечает свойство `flexWrap: 'wrap'`.

## Задания (Project 2 Step 5)

Весь код объединен в рамках единого приложения и разделен на несколько независимых компонентов для удобства переиспользования.

#### Компоненты:
- **GridLayout** (Task 1): Компонент для вывода данных в виде гибкой сетки. Умеет принимать количество колонок (columns) и отступы (spacing).
- **ResponsiveHeader** (Task 2): Умная шапка навигации. Сама определяет, на какой ОС она запущена, и добавляет необходимые отступы от края экрана (через `useSafeAreaInsets`), чтобы контент не "заезжал" под часы.
- **AdaptiveLayout** (Task 3): Основной каркас экрана. На телефонах выводит контент простым скролл-листом, а на планшетах (width >= 768) перестраивает интерфейс, добавляя Sidebar (боковое меню навигации).

**Ключевые концепции:**
- React Native StyleSheet API
- Мобильный Flexbox (где `flexDirection` по умолчанию — `column`)
- Динамические стили (передача массива стилей `style={[styles.base, isTablet && styles.tablet]}`)
