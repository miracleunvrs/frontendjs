## Почему React.ChangeEvent вместо any

Использование React.ChangeEvent<HTMLInputElement> дает:

- Проверку ошибок на этапе компиляции  
- Подсказки в IDE  
- Безопасный доступ к свойствам события  
- Предотвращение ошибок во время выполнения  

Использование any убирает типизацию и не рекомендуется.

---

## Когда нужны явные дженерики

Явные дженерики нужны, когда TypeScript не может автоматически вывести тип.  

Пример:

```ts
useState<User[]>([]);
TypeScript не может определить тип пустого массива без указания.

Если начальное значение известно, вывод типов достаточно:

useState("");
```
