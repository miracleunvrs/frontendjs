# Lab 4.2: Refactoring Article Manager
св
## 1. Информация о использовании AI
- Инструмент: ChatGPT / GPT-5-mini
- Дата: 2026-02-07

## 2. Конкретные промпты

### Промпт 1:
`Объясни, как разделить монолитный ArticleManager на ArticleList, ArticleItem и AddArticle`

Что получил: Подробное объяснение концепции рефакторинга: как вынести форму добавления статьи в отдельный компонент AddArticle, как создать ArticleList для списка статей и ArticleItem для отдельной статьи с локальным состоянием для показа/скрытия summary.

### Промпт 2:
`Сделай пример кода для всех компонентов с проверкой и обработкой событий`

Что получил: Полный пример кода с использованием useState для состояния формы, локального isOpened в ArticleItem, обработчиками onClickAdd, onClickRemove, onClickToggle и передачей данных через props.

### Промпт 3:
`Объясни, зачем нужен локальный state в ArticleItem и как его использовать`

Что получил: Объяснение, что локальное состояние isOpened позволяет ArticleItem управлять показом summary независимо от родительского компонента, улучшая производительность и делая компоненты переиспользуемыми.

### Промпт 4:
`Как можно сделать render props для ArticleManager`

Что получил: Инструкции по применению render props для добавления и списка статей, чтобы уменьшить зависимость ArticleManager от конкретных компонентов и повысить гибкость архитектуры.

## 3. Модификация и проверка
- Внедрил рекомендации AI в свой код.
- Проверил, что:
  - Добавление статьи работает корректно через AddArticle.
  - Удаление статьи работает через ArticleItem и ArticleList.
  - Показ/скрытие summary работает локально для каждого ArticleItem.
  - Render props (опционально) позволяют менять компоненты без изменения ArticleManager.

## 4. Что узнал
- Научился разносить монолитный компонент на мелкие, переиспользуемые.
- Понял, как передавать данные и события через props.
- Освоил локальное состояние для улучшения производительности.
- Ознакомился с render props для повышения гибкости компонентов.
- Убедился, что рефакторинг улучшает читаемость и поддержку кода.
