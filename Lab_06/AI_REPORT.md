## Lab 06.2: Динамические маршруты и loaders
---

## 1. Информация об использовании AI
- Инструмент: ChatGPT / Cursor
- Дата: 2026-02-21

## 2. Конкретные промпты

### Промпт 1:
`Как добавить динамический маршрут courses/:id в createBrowserRouter`
Что получил: Пример объявления маршрута с path: "courses/:id", element с компонентом страницы курса и объяснение, что id доступен в компоненте через useParams().

### Промпт 2:
`Объясни, как использовать loader для загрузки данных курса по id перед отрисовкой`
Что получил: Описание поля loader в конфиге маршрута: асинхронная функция с params, получение id из params.id, загрузка данных и return объекта; в компоненте — useLoaderData() для доступа к этим данным.

### Промпт 3:
`Покажи пример useSearchParams для сортировки списка с кнопкой`
Что получил: Пример использования useSearchParams(), get("sort"), setSearchParams({ sort: "asc" | "desc" }), кнопка переключает порядок и обновляет URL без перезагрузки страницы.

### Промпт 4:
`Как показать ошибку, если курс по id не найден, через errorElement`
Что получил: Добавление errorElement в маршрут с компонентом или JSX; при throw new Error() в loader React Router отображает errorElement вместо основного element.

## 3. Модификация и проверка
- Внедрил рекомендации AI в свой код.
- Проверил, что:
  - Страница списка курсов отображается с сортировкой по кнопке (asc/desc), URL содержит ?sort=asc или ?sort=desc.
  - Переход по ссылке на курс открывает страницу с данными курса (title, instructor, description).
  - При неверном id в URL отображается сообщение об ошибке (errorElement).
  - Данные курса приходят из loader, в компоненте используются useLoaderData и useParams.

## 4. Что узнал
- Динамические сегменты path: "courses/:id" и получение id через useParams().
- Loader для предзагрузки данных до рендера и useLoaderData() в компоненте.
- useSearchParams для чтения и изменения query-параметров (сортировка, фильтры).
- errorElement для отображения ошибок при throw в loader.
